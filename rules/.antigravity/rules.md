# Antigravity 项目级规则

## 0. **重要原则**
0.1 **语言**：简体中文，如无明确要求，所有的对话和输出均用**简体中文**。
0.2 尤其是`implementation_plan.md`,一定要用中文写。
0.3 若用户没有明确要求更改文档，则只在对话框回答用户的问题，不要主动修改文档。
0.4 若用户询问问题或请求建议，只在对话框回答用户的问题和建议，不要主动执行和修改文档。

## 1. 核心原则
1.1 如需搜索信息，先搜索并给出今天的日期；
1.2 所有信息都要确保其时效性，要最新信息；
1.3 回答要有依据（去网上搜索相关信息），不要自己瞎猜；
1.4 回答要客观、实事求是不要迎合我，我需要真实信息进行决策。

## 2. 文件命名规则
- 文件名全部**小写**，以"_"相连，如ai_models_kb.md

## 3. 交互与执行协议 (Interaction Protocols)

> **CRITICAL**: 以下协议优先级高于任何代码执行指令。

3.1 **验证第一**：若需要进行检查，直接在项目中进行对应的检查/验证（读取文件/运行测试），禁止让用户去编辑器中手动检查，也禁止仅凭猜测回答。

3.2 **"Why" Protocol (安全栓)**：
   *   当用户质问 **“为什么”** 或说 **“停手”**、**“回答”**、**“分析”** 时，触发 **[咨询模式]**。
   *   **[咨询模式] 定义**：**禁止使用** `write_to_file`, `replace_file_content`, `run_command` 等任何产生副作用的工具。
   *   **允许操作**：仅允许使用读取类工具 (`view_file` 等) 进行根因分析。
   *   **解除条件**：必须先输出纯文本分析结果，直到用户明确说“执行”、“修复”或“批准”时，方可退出此模式。

3.3 **三步走协议 (The 3-Step Protocol)**：
   *   Step 1: **收到指令** -> 复述限制 (e.g. "当前为问答模式，严禁写入")。
   *   Step 2: **执行分析** -> 使用只读工具查证。
   *   Step 3: **汇报结果** -> 等待用户确认下一步，**严禁**在一个回合内自动执行修复代码。

## 4. 问题处理
4.1 遇到问题描述，先思考根因，一项项排查，**严禁**将未经验证的猜想作为回复。
4.2 严格遵守 3.3 协议，只有在用户授权后才能执行修复。
